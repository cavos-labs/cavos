# Cavos Docs — PRODUCT.md

## Register
**product** (primary) — a developer documentation site for `@cavos/kit`. The reading/reference experience serves the content; the home page leans brand. Design serves comprehension and trust.

## Users & Purpose
- **Who:** developers (and AI agents) integrating Cavos device-native smart accounts on Starknet.
- **Context:** scanning for an API, copying a snippet, or feeding the whole site to an LLM. Often split-screen with an editor.
- **Job:** go from "what is this" → working integration fast. Trust the security claims (non-custodial, MPC-free).
- **Primary task per page:** read prose + copy a code block.

## Brand & Personality
Three words: **precise, technical, restrained.** Infrastructure-grade confidence without hype. Crisp, not soft.

## Brand system (from cavos-web)
- Electric-indigo `#402AFF` as the single accent on white. Ink `#0A0A0F`.
- Geist (sans) + Geist Mono. Letter-spacing −1% (-0.01em); display tighter.
- Light, monochrome-ink emphasis. Indigo ≤10% of surface — accents, links, current nav, code keywords only.
- Tight corner radii (3–8px). Crisp/technical, not rounded-friendly.
- Framed gridlines / hairline borders define structure. Subtle dot-grid texture allowed on hero.

## Anti-references
- Generic default Fumadocs neutral theme (what it looks like now).
- Dark hero panels, glassmorphism, gradient text, rounded SaaS-cream softness.
- Heavy indigo fills, neon, drenched color.

## Accessibility
Body text ≥4.5:1 on white. Indigo `#402AFF` on white ≈ 6.9:1 — safe for links/large text. Respect `prefers-reduced-motion`.
